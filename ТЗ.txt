«Платформа для обмена растениями PlantSwap»

1. ЦЕЛЬ ПРОЕКТА
Разработать веб-приложение на языке программирования Python с использованием микрофреймворка Flask, в котором пользователи могут публиковать объявления о передаче комнатных растений, загружать фотографии и бронировать (забирать) растения других участников.

2. ЗАДАЧИ ПРОЕКТА
Изучить возможности фреймворка Flask для создания веб-сервисов.
Реализовать систему регистрации и авторизации пользователей.
Реализовать защиту паролей с помощью хэширования.
Создать базу данных для хранения данных пользователей и информации о растениях.
Реализовать ORM-модели для работы с базой данных (User, Plant).
Реализовать веб-страницы с использованием шаблонизатора Jinja2.
Подключить Bootstrap 5 для создания адаптивного интерфейса.
Реализовать функционал добавления растений с заголовком, описанием и изображением.
Реализовать механизм бронирования («забора») растений другими пользователями.
Создать личный кабинет для просмотра своих лотов и забранных растений.
Обеспечить корректную работу приложения на локальном сервере.

3. ЭЛЕМЕНТЫ ПРИЛОЖЕНИЯ
3.1. Страница регистрации
Поле для ввода логина (username).
Поле для ввода пароля (password).
Кнопка «Создать аккаунт».
Ссылка на страницу входа для уже зарегистрированных пользователей.
Вывод сообщения, если логин уже занят.

3.2. Страница входа
Поле для ввода логина и пароля.
Кнопка «Войти».
Ссылка на страницу регистрации.
Вывод сообщения об ошибке при неверных данных.

3.3. Главная страница (Витрина)
Список всех доступных (не забранных) растений.
Карточка каждого растения: фото, название, описание и автор.
Кнопка «Забрать» для чужих растений.
Индикатор «Ваше фото» для собственных лотов пользователя.

3.4. Личный кабинет (Мои цветы)
Приветствие с именем пользователя.
Раздел «Я забрал»: список растений, забронированных текущим пользователем.
Раздел «Я выставил»: список собственных растений пользователя со статусом («Забрали» или «Ищет дом»).

3.5. Форма добавления растения
Поле для ввода названия цветка (обязательное).
Текстовое поле для описания.
Поле для выбора и загрузки файла изображения.
Кнопка «Опубликовать».

4. ХРАНЕНИЕ ДАННЫХ
4.1. База данных (SQLite)
Таблица User: id, username (уникальный), password_hash.
Таблица Plant: id, title, description, image_name, author, taker_id (ID пользователя, который забрал растение).
4.2. Файлы
Загруженные фотографии растений хранятся в папке static/uploads/.

5. ИНТЕРФЕЙС
Использование Bootstrap 5.
Адаптивная вёрстка для мобильных устройств и ПК.
Единый базовый шаблон (base.html) для всех страниц.
Навигационная панель с кнопками «Мои цветы», «Отдать цветок» и «Выйти».

6. ИСПОЛЬЗУЕМЫЕ ТЕХНОЛОГИИ
Python 3.x.
Flask.
SQLAlchemy (ORM).
SQLite.
Bootstrap 5.
Jinja2.
Werkzeug (для защиты паролей и обработки имен файлов).
